⬠ System Forge · Design Principles

UX foundations for a SysML2 modeling tool — reducing cognitive load in B2B engineering software

2025–26 · Collaboration with Cauê Napier (Systems Engineer, SSC)

Context

System Forge is a model-based systems engineering (MBSE) tool in early development. Cauê Napier, systems engineer at the Swedish Space Corporation (SSC), is building it as a side project to make SysML2-based modeling more accessible.

At this stage — core architecture taking shape — there's an opportunity to establish design principles before patterns solidify.

Framing and Reframing

Initial framing: Support the product with UX — review screens, improve layouts.

The reframe: Before reviewing screens, define what principles those screens should follow. SysML2 specifies semantics; presentation and interaction are left to tooling. That's the design space — mapping it was the first task.

Example Principles

Three examples showing the range — from product vision to learnability research:

Product Vision
01 · One Model, Many Views
Single source of truth with synchronized representations. Diagram, tree, table, text all reflect the same model. Graph view is the exception: provides overview, not a clickable navigation mode.
B2B UX Research
03 · Progressive Disclosure
Show what the current task requires, hide everything else. Complexity is available, never imposed. Ref: Nielsen Norman Group enterprise patterns.
Learnability
09 · Safe Exploration
Learners need to try things without risk. Generous undo, clear recovery paths, non-destructive defaults. Mistakes are cheap; hesitation is expensive.
To Validate
04 · Click Anywhere, Land Everywhere
Selecting an element in any view highlights it in all views. Note: May not apply to graph view. At minimum: clear indication of current location.

Next Steps

Workflow documentation — Map common tasks with principles applied. User testing — Validate assumptions with practitioners. UI component library — Build components that embody principles explicitly.

Reflection

On the work: SysML2 defines semantics; presentation is the tool's responsibility. Translating that into concrete guidance — "you can style this element, but this shape is fixed" — required working through the spec systematically.

On principles before pixels: The expectation for UX Designers is often visual output. But a dropdown without a principle is just a guess. Principles create constraints — and constraints make design decisions defensible, not arbitrary.
