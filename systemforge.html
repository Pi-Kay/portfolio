Reframing the Problem

Early in the project, it became clear that jumping to UI elements was premature. The first task was defining principles — what guides decisions, not what pixels look like.

The realization: You can't design a dropdown or a panel without knowing what principles that dropdown serves. Principles first, then patterns, then components.

This reframing surfaced four interconnected challenges:

Challenge 1
B2B UX in Specialized Domains
Standard UX heuristics assume consumer contexts. Specialized B2B requires understanding what practitioners actually do — not what generic best practices suggest.
Challenge 2
SysML2 Constraints
What's fixed by the standard? What's open to interpretation? This boundary isn't documented — it had to be mapped explicitly.
Challenge 3
Integration Opportunities
Learning from other complex professional tools — CAD, IDEs, 3D modeling — where has UX successfully reduced cognitive load in dense interfaces?
Challenge 4
Relearning Cost
Where is improvement worth the switching cost? Reducing time to learn SysML2, reducing cognitive load, reducing errors — which changes justify relearning?

Redefined Outcomes

With the problem reframed, the deliverables became clearer:

Outcome 1
10 Design Principles
Principles specific to System Forge — not generic UX guidelines. Each addresses a concrete source of cognitive load or navigation failure in MBSE tools.
Outcome 2
SysML2 Constraint Map
Explicit documentation of what's fixed by the standard (semantics, core notation) versus what's open for learnability improvements (layout, hierarchy, interaction).
Outcome 3
UI Pattern Categories
20 categories of UI decisions, prioritized by impact. Starting point for detailed component work — each category maps to specific principles.

UI Pattern Categories

Before designing individual components, map the territory. These 20 categories organize UI decisions by function — each links back to specific principles and SysML2 constraints.

Navigation & Orientation
  • Graph navigation controls
  • Zoom & pan behavior
  • Breadcrumb & location indicators
  • View switching & synchronization
  • Search & filter interfaces
Element Representation
  • Node styling & hierarchy
  • Edge display & density
  • Labels & annotations
  • Grouping & containment
  • State & status indicators
Interaction & Editing
  • Selection & multi-select
  • Drag, drop & connection
  • Context menus & quick actions
  • Undo/redo & history
  • Validation feedback
Density & Scale
  • Collapse & expand patterns
  • Level-of-detail switching
  • Minimap & overview
  • Filtering & hiding
  • Layout algorithms & spacing

Each category will be developed individually — mapping to principles, documenting SysML2 constraints, and defining pattern options with cognitive load tradeoffs.
